About This System
Physics-Informed Neural Networks for Load Forecasting

This platform uses a Physics-Informed Neural Network (PINN) trained on UCI's Household Electric Power Consumption dataset. Unlike black-box neural networks, PINNs embed physical laws directly into the loss function — ensuring predictions respect Ohm's Law (P = V·I), temporal smoothness, and power stability constraints.

LAYER 01
Data Ingestion
Reads Voltage, Current, Active Power from sensor streams or uploaded CSV. Handles missing values and normalization via MinMaxScaler.
LAYER 02
PINN Model
3-layer MLP (64→64→1) with Tanh activations. Inputs: [Voltage, Current, t]. Output: Global Active Power.
LAYER 03
Physics Loss
Penalizes deviation from P = V·I, enforces smooth temporal gradients (dP/dt ≈ 0), and stability between consecutive predictions.
LAYER 04
REST API
Flask backend exposes /predict, /forecast, /single endpoints. Supports both real PyTorch inference and physics simulation fallback.
Model ArchitecturePyTorch
class PINN(nn.Module): def __init__(self): super().__init__() self.net = nn.Sequential( nn.Linear(3, 64), # input: [Voltage, Current, t] nn.Tanh(), nn.Linear(64, 64), nn.Tanh(), nn.Linear(64, 1) # output: Global_active_power ) def forward(self, X, t): return self.net(torch.cat([X, t], dim=1)) # Physics-informed composite loss: loss = loss_data + 0.3 * (loss_smooth + loss_physics + 0.1 * loss_stability)
